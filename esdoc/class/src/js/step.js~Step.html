    public constructor(tour: Tour, options: Object): Step source

    Create a step

    Params:

    NameTypeAttributeDescription
    tour Tour

    The tour for the step
    options Object

    The options for the step
    options.attachTo Object | string

    What element the step should be attached to on the page. It can either be a string of the form [element] [on] (where [element] is an element selector path):

    const new Step(tour, {
  attachTo: '.some .selector-path left',
  ...moreOptions,
})'

    Or an object with those properties:

    const new Step(tour, {
  attachTo: { element: '.some .selector-path', on: 'left' },
  ...moreOptions
})'

    If you use the object syntax, element can also be a DOM element. If you don’t specify an attachTo the element will appear in the middle of the screen.
    options.attachTo.element HTMLElement | string
    options.attachTo.on string
    options.advanceOn Object | string

    An action on the page which should advance shepherd to the next step. It can be of the form "selector event":

    const new Step(tour, {
  advanceOn: '.some .selector-path click',
  ...moreOptions
})'

    ...or an object with those properties:

    const new Step(tour, {
  advanceOn: { selector: '.some .selector-path', event: 'click' },
  ...moreOptions
})'

    event doesn’t have to be an event inside the tour, it can be any event fired on any element on the page. You can also always manually advance the Tour by calling myTour.next().
    options.beforeShowPromise function

    A function that returns a promise. When the promise resolves, the rest of the show code for the step will execute.
    options.buttons Object[]

    An array of buttons to add to the step. These will be rendered in a footer below the main body text.
    options.buttons.button.action function

    A function executed when the button is clicked on
    options.buttons.button.classes string

    Extra classes to apply to the <a>
    options.buttons.button.events Object

    A hash of events to bind onto the button, for example {'mouseover': function(){}}. Adding a click event to events when you already have an action specified is not supported. You can use events to skip steps or navigate to specific steps, with something like:

    events: {
  click: function() {
    return Shepherd.activeTour.show('some_step_name');
  }
}
    options.buttons.button.text string

    The HTML text of the button
    options.classes string

    A string of extra classes to add to the step's content element.
    options.highlightClass string

    An extra class to apply to the attachTo element when it is highlighted (that is, when its step is active). You can then target that selector in your CSS.
    options.tippyOptions Object

    Extra [options to pass to tippy.js]https://atomiks.github.io/tippyjs/#all-options
    options.scrollTo boolean

    Should the element be scrolled to when this step is shown?
    options.scrollToHandler function

    A function that lets you override the default scrollTo behavior and define a custom action to do the scrolling, and possibly other logic.
    options.showCancelLink boolean

    Should a cancel “✕” be shown in the header of the step?
    options.showOn function

    A function that, when it returns true, will show the step. If it returns false, the step will be skipped.
    options.text string

    The text in the body of the step. It can be one of four types:

    - HTML string
- Array of HTML strings
- `HTMLElement` object
- `Function` to be executed when the step is built. It must return one of the three options above.
    options.title string

    The step's title. It becomes an h3 at the top of the step.
    options.when Object

    You can define show, hide, etc events inside when. For example:

    when: {
  show: function() {
    window.scrollTo(0, 0);
  }
}
